# PR Creator Skill - Changelog

## [2.0.0] - 2026-01-11

### 🎯 重大改进：智能版本判断

**背景**: 原有的简单规则（任何 `feat` → minor）过于激进，未能区分内部优化与面向用户的新功能。

### Changed

**版本判断逻辑升级**:
- ❌ 旧规则: `Any feat: → minor`
- ✅ 新规则: 基于**用户感知影响**判断
  - **Minor**: 至少 2 个用户可见的新功能
  - **Patch**: 单一 feat 的 UI 优化/内部改进

**判断原则**:
1. 关注用户感知的变化，而非开发者的提交类型
2. UI 优化、内部重构通常是 patch
3. 提供上下文建议，让用户最终决策

### Added
- 新增详细的版本判断示例（3 个场景）
- 新增中文描述和版本号
- 更清晰的 Capabilities 说明

### 示例对比

**场景: UI 布局优化 + Bug 修复**
```
提交:
- feat(ui): 优化推荐设置布局与视觉层次
- fix: 修复 LOCAL_STORAGE_KEYS 未定义错误
- docs: 添加策略集成文档
```

| 版本 | 旧判断 | 新判断 | 理由 |
|------|--------|--------|------|
| 1.x  | **minor** ❌ | **patch** ✅ | UI 调整是改进而非新功能 |

**场景: 多个独立新功能**
```
提交:
- feat: 添加 RSS 订阅管理
- feat: 实现文章自动翻译
```

| 版本 | 旧判断 | 新判断 | 理由 |
|------|--------|--------|------|
| 1.x  | **minor** ✅ | **minor** ✅ | 两个用户可见的新功能模块 |

### Migration Guide

如果你正在使用旧版本的 skill:

1. **更新 skill**:
```bash
openskills install wxy/pr-creator -y
openskills sync -y
```

2. **重新评估现有 PR 的版本号**:
   - 如果你的 PR 只包含 UI 优化/内部重构，建议降级为 patch
   - 如果包含多个用户可见的新功能，维持 minor

3. **调整工作流程**:
   - 在创建 PR 时，AI 会提供更合理的版本建议
   - 仍然可以手动调整版本等级
   - 注意检查 PR 描述中的版本提升理由
